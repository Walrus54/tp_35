# Практическое занятие: знакомство с библиотекой Qt

Реализация примеров **2.7**, **2.8** и **2.9** из методического пособия
«Создание пользовательских интерфейсов в программах на C++ с использованием
библиотеки Qt». Версия библиотеки — **Qt 5.15**.

## Структура веток

Каждая задача оформлена в **отдельной ветке** (как требует задание). В каждой
ветке лежит самостоятельное, независимо собираемое Qt-приложение.

| Ветка                 | Пример | Приложение                                                              |
| --------------------- | ------ | ----------------------------------------------------------------------- |
| `main`                | —      | README, .gitignore, текст задания                                       |
| `example-2.7-square`  | 2.7    | «Возведение числа в квадрат» — создание новых слотов, валидатор ввода   |
| `example-2.8-counter` | 2.8    | «Счётчик» — генерация собственного сигнала                              |
| `example-2.9-figures` | 2.9    | «Движущиеся изображения» — события таймера, перерисовки, показа/скрытия |

## Сборка (любая ветка с приложением)

```bash
git checkout example-2.7-square      # либо -counter / -figures
cmake -S . -B build
cmake --build build
./build/<имя_цели>                   # square / counter / figures
```

Требуется установленный Qt 5.15 (компонент Widgets) и компилятор C++17.

## Замечание о кодировке

Исходный код примеров взят из методички, где для русификации интерфейса
используется `QTextCodec::codecForName("Windows-1251")`. В оригинале (Qt4)
исходные файлы хранились в кодировке **Windows-1251**, поэтому строковые
литералы корректно декодировались `codec->toUnicode(...)`.

Чтобы сохранить этот подход в Qt 5.15, файлы с исходным кодом (`*.h`, `*.cpp`)
хранятся в кодировке **Windows-1251**, а в `CMakeLists.txt` компилятору
передаются флаги `-finput-charset=windows-1251 -fexec-charset=windows-1251`.

## Отличия от кода методички (адаптация под Qt 5.15)

- `#include <QtGui>` заменён на `#include <QtWidgets>`: в Qt5 классы виджетов
  вынесены в отдельный модуль `QtWidgets`.
- `QTextCodec` подключается явно через `#include <QTextCodec>` (модуль `QtCore`).
- Сборка переведена с qmake на **CMake** с `CMAKE_AUTOMOC`.
- Исправлены опечатки оригинала (например, `culcbutton` → `calcbutton`).
- Добавлены include-guard и виртуальный деструктор там, где это требуется.
